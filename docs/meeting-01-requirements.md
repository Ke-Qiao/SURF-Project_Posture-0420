# SURF第一次会议要求总结

## 1. 项目基本信息

- 项目名称：AI-Based Human Posture Detection and Correction System Using Computer Vision
- SURF 编号：SURF-2026-0420
- PI：Gomesh Nair
- 项目定位：基于计算机视觉的人体姿态检测与纠正系统
- 最终展示形式：SURF Poster Fair 海报展示

## 2. 会议核心结论

### 2.1 项目研究对象是 `posture`，不是 `pose`

- 本项目研究的是站立状态下的人体体态 `posture`，不是动作姿势 `pose`。
- 老师特别强调不要把“挥手、抬手、做动作”这类姿势识别误当成项目目标。
- 当前阶段只关注人体站立时是否存在不良体态，例如头前伸、圆肩、躯干倾斜等。

### 2.2 当前优先范围是站姿，且以侧视图为主

- 只做 standing posture，不做 sitting posture。
- 只要求站姿，不要求坐姿检测。
- 图像视角优先采用侧视图 side view / sideways。
- 原因是正面视图对轻微倾斜不敏感，侧视图更容易观察头部、肩部、髋部、膝部是否偏离理想直线。

### 2.3 检测逻辑是关键点加角度判定

- 需要在人体图像中获取关键点，例如耳朵、肩膀、躯干、髋部、膝盖，必要时可延伸到脚踝。
- 基于关键点构建 skeleton。
- 通过关键点连线计算角度或偏移量，判断姿态是否良好。
- 老师给出的直观标准是：耳-肩-髋-膝等关键点应尽量接近一条理想直线；若出现明显偏离、前倾、弯曲、塌陷，则判为 bad posture。
- 系统不只要给出 good / bad，还鼓励输出具体倾斜角度或偏移程度。

## 3. 数据集与图像要求

### 3.1 数据来源

- 可以混合使用一手数据和二手数据。
- 一手数据：自己或朋友站立拍照。
- 二手数据：网络图片、公开数据集。
- 老师已经提供了参考数据和 Kaggle 数据集线索。

### 3.2 数据筛选标准

- 优先使用 full-body 图像，即从头到脚尽量完整可见。
- 优先使用侧视站姿图。
- 避免大量坐姿图、遮挡严重图、关键部位缺失图。
- 避免将不适合本项目目标的数据直接用于训练，否则会带来偏差。

### 3.3 数据处理要求

- 需要进行预处理和筛选。
- 需要进行关键点标注，或基于现成关键点数据进行训练。
- 如现有样本不足，可以考虑 synthetic dataset augmentation。
- 老师允许团队自行补拍数据，以提升对目标问题的贴合度。

## 4. 模型、工具与技术路线

### 4.1 老师给出的基线方向

- 可使用 OpenCV。
- 可使用 MediaPipe 进行人体关键点检测。
- 可使用 YOLO 系列模型，老师在材料中提到 YOLOv8 方向。
- 老师现场展示的是一个基于现成库快速搭起来的 skeleton 检测基线，并明确表示这个版本还不够准确，需要后续改进。

### 4.2 老师提供的基线代码信息

- 参考代码使用了 `cv2` 和 `mediapipe`。
- 代码通过摄像头实时读取画面。
- 代码根据肩膀高低差和上半身倾斜程度做简单分类。
- 这个版本只能作为 baseline，不是最终成果。
- 你们的任务不是机械复现，而是在此基础上逐步提高检测质量、鲁棒性和可解释性。

### 4.3 当前建议的技术理解

- 第一层：先跑通老师给的实时骨架检测基线。
- 第二层：明确适合项目的数据集，并完成筛选与训练准备。
- 第三层：让模型能更稳定地区分 good posture 和 bad posture。
- 第四层：加入角度量化或多关键点联合判定。

## 5. 评估指标与结果要求

- 不能只做视觉演示，还要有机器学习评估结果。
- 会议明确提到要输出标准指标，例如：
  - F1 Score
  - Precision
  - Recall
  - Mean Average Precision `mAP`
- 这些指标不仅用于技术验证，也会服务于 SURF 海报与研究展示。

## 6. 系统交付要求

### 6.1 功能层面

- 需要实现姿态检测能力。
- 允许两种交付路径：
  - 实时摄像头检测
  - 视频录制后检测
- 老师更倾向于看到可直接演示的系统。
- 系统最好能在检测 bad posture 后给出量化结果，而不是只显示一个标签。

### 6.2 环境兼容性

- 代码必须能在中国大陆网络环境下运行。
- 不能依赖必须翻墙才能稳定使用的服务。
- 老师明确提到要在关闭 VPN 的条件下验证运行情况。
- 后续方案若依赖外部模型或下载源，需要优先考虑中国环境兼容性。

### 6.3 研究展示层面

- 项目最终需要服务于海报展示。
- 海报中应覆盖研究问题、摘要、方法、实验结果等要素。
- 老师已提供项目简介和可参考的研究框架材料。

## 7. 时间线与阶段任务

结合老师提供的时间线图片，项目大体分为五个阶段：

### 7.1 Week 1-2: Learning and Training

- 学习 AI、计算机视觉、深度学习基础
- 熟悉 Python、OpenCV、YOLO 等工具
- 理解研究伦理与数据处理基础

### 7.2 Week 2-3: Data Collection and Annotation

- 收集姿态图像
- 进行数据清洗和过滤
- 完成关键点或相关标注准备

### 7.3 Week 4-6: Model Development

- 训练姿态检测模型
- 调整参数
- 比较不同配置下的性能

### 7.4 Week 7-8: System Integration

- 接入摄像头或实时检测界面
- 加入基于规则的姿态纠正或提醒逻辑
- 视团队能力决定是否考虑嵌入式部署

### 7.5 Week 9-10: Evaluation and Reporting

- 输出评估指标
- 整理实验结果
- 完成海报与展示材料

## 9. 从会议中提炼出的明确约束

- 项目当前只做站姿，不做坐姿。
- 项目当前优先做侧视，不把正视图作为主路线。
- 图像尽量使用全身图。
- 检测逻辑应围绕关键点、骨架和角度展开。
- 基线代码只是起点，不是最终方案。
- 结果需要既能演示，也能量化评估。
- 最终成果需要能用于 SURF 海报展示。
- 技术方案需要考虑中国大陆网络兼容性。
